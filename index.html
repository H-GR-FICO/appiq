Filozofie platformy Platform Philosophy

Proč vznikla HOPI AppIQ? Why HOPI AppIQ exists?

Původní záměr, který prostupuje celou architekturou — kostra každého rozhodnutí. The original intent running through the entire architecture — the backbone of every decision.
Původní záměr: Hledáme platformu pro rychlý vývoj, nasazení a distribuci aplikací vyvíjených firmou a jejími uživateli za pomoci umělé inteligence. Každé architektonické rozhodnutí se hodnotí prizmatem: „Umožňuje toto naší firmě rychleji stavět, nasazovat a sdílet aplikace s AI?" Original Intent: We are looking for a platform for rapid development, deployment and distribution of applications built by the company and its users with AI. Every architectural decision is evaluated through: "Does this enable our company to build, deploy and share applications with AI faster?"
1
Tvorba Build
HOPIQ AI snižuje bariéru vstupu na minimum. Analytik navrhne aplikaci bez znalosti programování. AI Configurator převede zadání na funkční modul. HOPIQ AI lowers the barrier to a minimum. An analyst can design an application without programming. AI Configurator converts requirements into a functional module.
2
Nasazení Deploy
CI/CD pipeline, Azure Static Web Apps, Entra ID SSO. Od kódu k produkci bez manuálního zásahu. Distribuce přes Teams, prohlížeč, SharePoint. CI/CD pipeline, Azure Static Web Apps, Entra ID SSO. From code to production without manual intervention. Distribution via Teams, browser, SharePoint.
3
Škálování Scale
Jednou postavené jádro: Finance → Operations → Purchasing → HR → IT → celá skupina. Každá oblast dostane HOPI AppIQ | [Oblast]. Bez přepisu kódu. Core built once: Finance → Operations → Purchasing → HR → IT → entire group. Each area gets HOPI AppIQ | [Area]. No code rewrite.

⏩ Filtrování & řazení

Filtry ALL / WEB / APP omezí zobrazený stream. Kliknutím na záhlaví sloupce (↑/↓) seřadíte dle ID, Názvu, Priority, Statusu, Domény, Modulu nebo řešitele. Klik znovu = otáčí pořadí.

🏠 Doména / Modul / Komponenta

Doména = funkční oblast dle architektury (Studio, Finance, Platform…). Modul = standardní enum z Module Registry. Komponenta = volný popis konkrétního prvku. Editovat přes ✎ — modul se filtruje podle vybrané domény.

Ve ⚙️ Nastavení můžete zapnout/vypnout automatické řazení, skrýt CLOSED nebo nastavit výchozí filtr při načtení.

⌨️ Zkratky

Esc zavřít panel  |  Enter v poli Název = přidat úkol  |  klik na ✎ = editovat poznámku

🔢 Struktura úkolu

Název = krátký identifikátor (jako Jira summary, max ~60 znaků). Popis = detailní specifikace co, jak a proč — editovat přes ✎ (textarea). Hover nad názvem v tabulce zobrazí popis. Komentář = krátká provozní poznámka / blokátor.

Formát ID: AIQ-NNNNN (5 míst, Jira standard) — unikátní, sekvenční. Odkazujte jako “AIQ-00001 — Název”. OIL = název metodiky a panelu.
